# Jarvis Benchmark Suite

This document describes the two bootstrap suites that replaced all previous simulated/fake benchmark data.

> [!IMPORTANT]
> All previous benchmark files (`ANTHROPIC_STYLE_REPORT.json`, `QUANTITATIVE_BENCHMARKS.json`) contained **hardcoded fake scores** and have been deleted. Every score in this system is generated by **real task executions** with **automated correctness verification**.

---

## Suite 1: System Analysis Bootstrap

**Purpose**: A complete, reproducible snapshot of the live Jarvis system state.

**Command**:
```bash
make system-analysis
```

**Produces**:
- `benchmarks/results/system_report.json` — machine-readable full state
- `benchmarks/results/system_report.md` — human-readable summary table

| Check Script | What it measures |
|---|---|
| `check_hardware.py` | CPU model, RAM/swap, ZRAM, disk layout, NixOS detection, GPU VRAM |
| `check_services.py` | `systemctl --user` status, uptime for all 8 Jarvis services |
| `check_models.py` | Ollama inventory, model sizes, routing role validation |
| `check_knowledge.py` | SQLite chunk counts, category distribution, inbox depth, orphan detection |
| `check_security.py` | Pending grants, session token age, secrets dir permission check |
| `check_api_keys.py` | Key presence per provider (Anthropic, OpenAI, Google, etc.) — values never printed |

## Suite 2: Honest Benchmark Suite (Anthropic Standards)

**Purpose**: Rigorous, honest evaluation following [Anthropic's evaluation methodology](https://www.anthropic.com/research) — not just its aesthetic style.

### Honesty Guarantees

| Principle | Implementation |
|---|---|
| Automated verifier | Every task has a deterministic correctness check — no human judgment |
| No cherry-picking | All tasks in every category always run in every report |
| Temperature = 0 | Fixed seed=42. No sampling variation in scores |
| Frontier comparison | Same tasks run against both local and cloud models side-by-side |
| No_key graceful | Cloud categories produce `status: no_key` (not an error) when key absent |
| Held-out verification | Task prompts are checked to not appear in any indexed knowledge base |

### Commands

```bash
make benchmark           # Run all categories against default local model
make benchmark-offline   # Local Ollama models only (no API keys needed)
make benchmark-online    # Cloud models only (API keys required)
make benchmark-full      # Full matrix: all models (local + cloud)
```

Alternatively, run a specific category against a specific model:
```bash
python benchmarks/eval/runner.py --category factual --offline
```

### Evaluation Categories

| Category | Tasks | Scoring Method |
|---|---|---|
| **Coding** | 8 (5 Python, 2 Rust, 1 Nix) | `exec()` + unit test assertions |
| **Instruction Following** | 20 | Automated constraint checks (word count, JSON structure, regex, banned words) |
| **Factual / MMLU** | 30 | Letter extraction match (A/B/C/D). Domain: Linux, NixOS, AI/ML, Security |
| **Agentic Tool Use** | 5 | Tool call log sequence + output constraint verification |
| **RAG Accuracy** | 15 (10 indexed + 5 not-indexed) | Keyword presence + hallucination detector |

### Output

Each run produces:
- `benchmarks/results/benchmark_<run_id>.json` — full per-task results
- `benchmarks/results/benchmark_latest.json` — symlink to most recent run
- `benchmarks/results/benchmark_report.md` — human-readable summary

### Report Format (sample)

```
| Category              | Model                    | Type    | Pass | Total | Pass Rate |
|-----------------------|--------------------------|---------|------|-------|-----------|
| coding                | qwen2.5-coder:14b        | 🖥️ local | 6    | 8     | 75.0%     |
| instruction_following | qwen2.5-coder:14b        | 🖥️ local | 17   | 20    | 85.0%     |
| factual               | qwen2.5-coder:14b        | 🖥️ local | 22   | 30    | 73.3%     |
| coding                | claude-3-haiku-20240307  | ☁️ cloud | –    | –     | skipped (no key) |
```

### Adding Cloud Model Keys

Once you have API keys, add them with:
```bash
jarvis set-key anthropic <KEY>
jarvis set-key openai <KEY>
```
Then run `make benchmark-online` or `make benchmark-full`. No code changes needed.

## Deleted Files

The following files were deleted because they contained **simulated/fake data** and would mislead anyone reading them:

| File | Reason |
|---|---|
| `docs/ANTHROPIC_STYLE_REPORT.json` | Hardcoded fake scores (e.g., "94.2% HumanEval") |
| `docs/QUANTITATIVE_BENCHMARKS.json` | Simulated synthesis ratios |
| `docs/MANPAGE.md` | Duplicate of `jarvis.1` |
| `tests/agentic_metrics_runner.py` | Self-admitted simulation code |
| `tests/benchmark_suite.py` | Measured latency only, no correctness verification |
| `tests/benchmarks/benchmark_runner.py` | No correctness scoring |
